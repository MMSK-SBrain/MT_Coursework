## Module Flow

### Week 1: Planning & Design (6-9 hours)

**Session 1: Project Selection (2-3h)**
- Choose from 5 curated options or propose custom
- Write project proposal
- Get instructor approval

**Session 2: Data Collection & EDA (3-4h)**
- Gather datasets
- Exploratory data analysis
- Document insights

**Session 3: Architecture Design (2-3h)**
- System architecture diagram
- Technology stack selection
- ML pipeline design

**Deliverable:** Approved proposal + EDA report + Architecture doc

---

### Week 2: Implementation (18-22 hours)

**Session 4: Model Development (6-8h)**
- Build preprocessing pipeline
- Train multiple models
- Compare and select best

**Session 5: Optimization & Testing (4-6h)**
- Hyperparameter tuning
- Error analysis
- Edge case testing

**Session 6: Deployment (6-8h)**
- Build API (Flask/FastAPI)
- Create UI (Streamlit)
- Deploy to cloud
- Obtain public URLs

**Deliverable:** Deployed application (API + UI) + Trained models

---

### Week 3: Polish & Present (12-16 hours)

**Session 7: Documentation (4-5h)**
- Comprehensive README
- API documentation
- Code comments
- User guide

**Session 8: Presentation Prep (4-5h)**
- Create slides
- Record demo video
- Practice presentation

**Session 9: Final Review (4-6h)**
- Code review and cleanup
- Final testing
- Peer review
- Submission

**Deliverable:** Complete project package + Presentation

## Technical Requirements

### Minimum ML Requirements
- ✅ Use 3+ different ML techniques
- ✅ Include supervised learning (classification or regression)
- ✅ Include unsupervised learning OR deep learning
- ✅ Achieve meaningful performance metrics

### Deployment Requirements
- ✅ Working API deployed to cloud (Heroku, Railway, Render, etc.)
- ✅ User-friendly interface deployed (Streamlit Cloud, Netlify, etc.)
- ✅ Both publicly accessible with working URLs
- ✅ Proper error handling and validation

### Documentation Requirements
- ✅ Comprehensive README.md
- ✅ API documentation (all endpoints)
- ✅ Code comments and docstrings
- ✅ User guide

### Presentation Requirements
- ✅ Professional slide deck (10-15 slides)
- ✅ Demo video (2-3 minutes)
- ✅ Live presentation (5-7 minutes + Q&A)

### Total: 200 Points

**Technical Implementation (100 points)**
- Problem definition & dataset (20)

- Model development (40)
- Code quality (15)
- Testing (15)

**Documentation & Presentation (60 points)**
- Documentation (30)
- Presentation quality (20)
- Peer review (10)

**Innovation & Impact (40 points)**
- Technical innovation (15)
- Real-world applicability (15)
- Portfolio quality (10)

**Grading Scale:**
- A: 180-200 points (Exceptional)
- B: 150-179 points (Good to Very Good)
- C: 120-149 points (Acceptable)
- Below 120: Does not meet requirements

## Instructor Notes

### Teaching Approach
- **Week 1:** Close guidance on scope and planning
- **Week 2:** Available for technical questions
- **Week 3:** Feedback on drafts, presentation coaching

### Key Checkpoints
- End of Week 1: Approve proposals, verify data access
- End of Week 2: Verify deployment working
- End of Week 3: Final presentations

### Common Pitfalls to Watch For
- Scope too ambitious (intervene early)
- Data availability issues (require verification)
- Deployment left until last day (encourage early deployment)
- Minimal documentation (provide templates)

### Flexibility
- Project options are guidelines, not rigid requirements
- Allow students to pivot if early issues arise
- Focus on learning and shipping, not perfection
