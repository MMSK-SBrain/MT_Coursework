# Module 10: Capstone Project - Detailed Rubric

**Total Points:** 200

## Section 1: Technical Implementation (100 points)

### 1.1 Problem Definition & Dataset (20 points)

**Problem Definition (10 points)**
- **Excellent (9-10):** Clear, compelling problem statement with real-world relevance. Well-defined scope and success criteria.
- **Good (7-8):** Clear problem statement with defined scope. Success criteria present.
- **Adequate (5-6):** Problem statement present but lacks clarity or relevance.
- **Poor (0-4):** Vague or missing problem statement. Unclear scope.

**Dataset Quality (10 points)**
- **Excellent (9-10):** Appropriate dataset, well-documented sources, 1000+ samples, proper licensing verified.
- **Good (7-8):** Appropriate dataset, documented, adequate size.
- **Adequate (5-6):** Dataset present but quality issues or small size.
- **Poor (0-4):** Inappropriate or inadequate dataset.

---

### 1.2 Data Preprocessing & EDA (30 points)

**Exploratory Data Analysis (15 points)**
- **Excellent (13-15):** Comprehensive EDA with 10+ visualizations, clear insights documented, thorough analysis.
- **Good (10-12):** Good EDA with 5-9 visualizations, insights documented.
- **Adequate (7-9):** Basic EDA with minimal visualizations.
- **Poor (0-6):** Minimal or missing EDA.

**Data Preprocessing (15 points)**
- **Excellent (13-15):** Proper handling of missing values, outliers, scaling, feature engineering. Well-documented pipeline.
- **Good (10-12):** Good preprocessing with most issues handled.
- **Adequate (7-9):** Basic preprocessing, some issues remain.
- **Poor (0-6):** Inadequate or missing preprocessing.

---

### 1.3 Model Development (40 points)

**Multiple Models Tried (10 points)**
- **Excellent (9-10):** 3+ algorithms compared systematically with justification.
- **Good (7-8):** 2-3 algorithms tried.
- **Adequate (5-6):** 1-2 algorithms, minimal comparison.
- **Poor (0-4):** Single model, no comparison.

**Model Selection & Tuning (10 points)**
- **Excellent (9-10):** Systematic hyperparameter tuning (GridSearch/RandomSearch), well-justified model selection.
- **Good (7-8):** Some tuning performed, reasonable selection.
- **Adequate (5-6):** Minimal tuning, unclear selection rationale.
- **Poor (0-4):** No tuning or selection process.

**ML Technique Diversity (10 points)**
- **Excellent (9-10):** Uses 3+ different ML techniques from multiple modules. Includes supervised + (unsupervised OR deep learning).
- **Good (7-8):** Uses 3 techniques, meets minimum requirements.
- **Adequate (5-6):** Uses 2-3 techniques, marginally meets requirements.
- **Poor (0-4):** Fewer than 3 techniques or missing required types.

**Model Performance (10 points)**
- **Excellent (9-10):** Exceeds target metrics significantly. Excellent for problem type.
- **Good (7-8):** Meets target metrics.
- **Adequate (5-6):** Approaching target metrics.
- **Poor (0-4):** Below target metrics or no improvement over baseline.
