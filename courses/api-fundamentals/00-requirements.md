# Course Requirements: API Fundamentals

**Created:** 2026-01-28
**Status:** Finalized

## 1. Course Context
- **Topic:** APIs (Application Programming Interfaces)
- **Industry/Domain:** Software Development / IT (for non-technical professionals)
- **Duration:** 2 hours (single session workshop)
- **Programming Language:** Python (using `requests` library)
- **Description:** A 2-hour workshop teaching non-technical professionals how to understand and consume APIs. Focus on practical skills for working with technical teams.
- **Format:** Guided practice (learners follow along on their machines)
- **Primary Goal:** Depth in consuming APIs - conceptual understanding + hands-on skills

## 2. Target Audience
- **Primary Learners:** Non-technical staff (business analysts, product managers)
- **Experience Level:** Complete beginners to programming
- **Prerequisites:** Basic computer literacy, laptop with Python installed
- **Job Role:** Business analysts, product managers, non-dev staff who collaborate with technical teams

## 3. Learning Outcomes

### What learners will DO:
1. Explain what an API is and why it matters for their work
2. Read and interpret API documentation
3. Make API calls using Python to fetch real data
4. Parse and use JSON response data
5. Understand API authentication basics (API keys)

### Critical Skills:
- Understanding the request/response cycle
- Making GET requests to public APIs
- Reading JSON data
- Using API keys for authentication

### Nice-to-have (if time permits):
- POST requests (sending data)
- Error handling basics
- Rate limiting awareness

## 4. Content Structure

### Main Modules:
1. **API Fundamentals** (~30 min) - Theory + concepts
2. **Consuming APIs** (~60 min) - Hands-on practice - PRIMARY FOCUS
3. **API Creation & Security Overview** (~20 min) - Conceptual awareness
4. **Wrap-up & Q&A** (~10 min)

### Sequencing:
- Theory first (what & why), then hands-on (how)
- Use real public weather API (OpenWeatherMap or similar)

### Theory/Practice Ratio:
- 40% Theory / 60% Hands-on practice

## 5. Assessment
- **Types:** No formal assessment (workshop format)
- **Success Criteria:** Learner successfully makes an API call and retrieves weather data by end of session

## 6. Constraints
- **Equipment:** Learners need laptops with Python 3 installed
- **Technical Requirements:** Internet access, Python `requests` library
- **Practice API:** Public weather/data API (OpenWeatherMap, JSONPlaceholder as backup)
