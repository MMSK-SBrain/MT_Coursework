## 🎯 What are Story Frameworks?

### **NOT:**

### **YES:**

- ✅ Instructor scripts with professional tone
- ✅ Progressive learning journeys (simple → complex)
- ✅ Integration of technical content with pedagogical storytelling

### **Think of them as:**
"**Professional technical lectures with a clear narrative arc**"

## 📖 Framework Structure

Each framework follows the **Teacher-Led Story Arc**:

### **1. SETUP (Slides 1-5, ~10 min)**
- **Hook students** with relevant question
- **Connect** to previous learning
- **Establish** why today's topic matters
- **Preview** the learning journey

### **2. TRIGGER (Slides 6-7, ~5 min)**
- **Present challenge** or real-world problem
- **Introduce** key concepts needed to solve it
- **Motivate** learning through practical need

### **3. RISING ACTION (Slides 8-24, ~50-55 min)**
- **Build understanding** systematically (3-4 parts)
- **Introduce concepts** one at a time
- **Provide examples** and calculations
- **Include practice** problems for active learning

### **4. CLIMAX (Slides 25-27, ~10-12 min)**
- **Integrate** all concepts learned
- **Solve** comprehensive problem
- **Demonstrate** real-world application
- **Show mastery** is achievable

### **5. RESOLUTION (Slides 28-30, ~10 min + Q&A)**
- **Summarize** key takeaways
- **Connect** to next session
- **Assign** practice work
- **Address** questions

## 🎓 Framework Philosophy

### **Teacher-Led Narrative Principles:**

#### **First-Person Instructor Voice**
```
❌ "The force can be calculated using..."
✅ "Let me show you how to calculate this force..."
```

#### **Personal Connection**
```
❌ "This topic is important"
✅ "I remember when I first learned this - it changed how I thought about vehicles"
```

#### **Collaborative Language**
```
❌ "You should understand..."
✅ "Walk with me through this calculation..."
```

#### **Build-Up to Confidence**
```
❌ "This is complex"
✅ "You already have everything you need - let's apply what you know"
```

### **Professional Balance:**

| Aspect | How We Handle It |
|--------|------------------|
| **Technical Rigor** | Never compromised - all formulas, units, and calculations are accurate |
| **Storytelling** | Used for structure and delivery, NOT for content creation |
| **Examples** | Real vehicles, real numbers, industry-relevant scenarios |
| **Tone** | Conversational yet authoritative - professor, not entertainer |
| **Visuals** | Engineering diagrams, technical schematics, professional graphs |

## 📊 PPT Creation Guidance

### **Slide Design Principles:**

#### **1. Professional Aesthetic**
- Clean layouts, consistent templates
- Engineering color scheme (blues, grays, orange for highlights)
- High-quality technical diagrams
- Readable fonts (equations in large, clear type)

#### **2. Visual Hierarchy**
- **Emphasized slides**: Formulas, complete examples, summaries
- **Supporting slides**: Explanations, context, practice problems
- **Reference slides**: Tables, comparison charts, specifications

#### **3. Animation Strategy**
- **Build animations**: For step-by-step calculations
- **Highlight**: Key terms as mentioned
- **Reveal**: Solutions after student practice pause
- **Minimal transitions**: Professional, not distracting

#### **4. Content Density**
- **Title slides**: Minimal text, strong visual
- **Explanation slides**: Diagrams + key points (not paragraphs)
- **Calculation slides**: Full worked examples with units
- **Summary slides**: Clean lists or flowcharts

### **Required Visual Elements:**

| Slide Type | Visual Elements |
|------------|----------------|
| **Setup (Hook)** | Strong opening image, relatable scenario |
| **Formulas** | Large, annotated equations with unit labels |
| **Calculations** | Step-by-step math with highlighting |
| **Diagrams** | Technical schematics, free-body diagrams, cross-sections |
| **Comparisons** | Side-by-side images or tables |
| **Summary** | Clean flowcharts or bullet lists |

## ⏱️ Timing Guidelines

### **Standard 2-Hour Session:**

| Section | Time | Slides | Notes |
|---------|------|--------|-------|
| **Setup** | 8-10 min | 1-5 | Don't rush the hook |
| **Trigger** | 5 min | 6-7 | Set up the challenge |
| **Rising Action** | 50-55 min | 8-24 | Core content (3-4 parts) |
| **Climax** | 10-12 min | 25-27 | Integration and application |
| **Resolution** | 10 min | 28-30 | Summary and next steps |
| **Q&A** | 30 min | - | Discussion and clarification |
| **Total** | 120 min | ~30 slides | 90 min lecture + 30 min Q&A |

### **Flexibility Points:**

**If Running Short:**
- Condense one practice problem
- Move detailed explanation to Q&A
- Assign one example as homework

**If Running Long:**
- Skip optional comparison slide
- Combine summary elements
- Move advanced topic to next session

**Cannot Skip:**
- Core formulas and methods
- Main worked examples
- Integration/climax moment
- Summary of key takeaways

## 💡 Pedagogical Best Practices

### **1. Build Progressive Confidence**

```
Session 1: "You already know Newton's laws - let's apply them to vehicles"
  ↓
Session 2: "Now that you can calculate forces, let's see how tires provide them"
  ↓
Session 3: "With forces and friction understood, we can calculate power needs"
  ↓
Session 4: "Knowing power requirements, let's see how engines generate it"
```

### **2. Connect Every Session**

**Always include:**
- "Last session, we learned [X]"
- "Today, we build on that with [Y]"
- "Next session, we'll see how [Z] uses what we learn today"

### **3. Use Real Examples**

**Good:** Honda Civic (1,300 kg, C_d = 0.30) at 100 km/h
**Better:** "The car many of you drive" with actual specs
**Best:** Local examples students can verify themselves

### **4. Make Math Accessible**

**For every formula:**
- Explain **what** it calculates
- Define **every variable** with units
- Show **worked example** immediately
- Provide **practice** problem

### **5. Emphasize Application**

**After every concept:**
- "This is why [real-world observation]"
- "Engineers use this to [practical application]"
- "[Company/Industry] applies this when [scenario]"

### **Q: What if I prefer different narrative style?**
**A:** The structure works with various styles - adjust scripts to match your voice.

### **Q: Do I need to follow slide numbers exactly?**
**A:** No - they're estimates. Adjust based on content depth and pace.

### **Q: Can I combine/split sections?**
**A:** Yes, if it serves pedagogy. Keep the 5-part arc intact for coherence.

**A:** Add more calculation slides in Rising Action. Template indicates where.
