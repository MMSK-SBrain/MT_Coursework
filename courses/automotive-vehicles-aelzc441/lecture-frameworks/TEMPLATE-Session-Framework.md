   - Adjust number of Parts in Rising Action based on # of concepts
   - Add formulas, diagrams, and examples from technical references

4. **Adapt the structure**:
   - More numerical sessions: Add more calculation slides
   - More conceptual sessions: Add more comparison/analysis slides
   - System-heavy sessions: Add component diagrams and operation sequences

5. **Maintain the Teacher-Led Narrative**:
   - Use first-person ("Let me show you...")
   - Include personal connections where appropriate
   - Build progressive understanding (simple → complex)
   - End with confidence and capability

6. **Professional Balance**:
   - Technical accuracy is paramount
   - Storytelling is the delivery method, not the content
   - Use narrative to make technical content accessible
   - Maintain engineering rigor

## 🎓 FRAMEWORK PHILOSOPHY

This template follows the **Teacher-Led Story Arc** principles:

**Setup → Trigger → Rising Action → Climax → Resolution**

But adapted for **professional technical education**:

- **Setup**: Context and relevance, not fictional backstory
- **Trigger**: Real engineering challenges, not dramatic conflict
- **Rising Action**: Building technical knowledge systematically
- **Climax**: Integration and application of concepts
- **Resolution**: Mastery confirmation and connection to future

**Key difference from entertainment storytelling:**
- The "story" is the learning journey
- The "characters" are concepts and systems
- The "conflict" is technical challenges
- The "resolution" is student competence

**Result**: Engaging, memorable lectures that build both knowledge and confidence.
